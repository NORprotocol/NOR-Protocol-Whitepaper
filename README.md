## ◆ 1. 摘要｜协议定位
NOR Protocol 是一个由智能合约驱动的 Web3 参与与激励协议。  
不同于以叙事为核心或依赖人为裁量的系统，NOR 采用规则优先（rule-first）的架构：  
参与条件、资产行为、激励分配与释放机制均在协议层一次性定义，并由智能合约自动执行。  
协议中不存在人为裁量、临时规则调整或选择性干预。  
所有参与者均在同一套确定性的系统逻辑下运行。  

📌 本白皮书用于解释 NOR Protocol 的设计动机与系统结构。  
📌 所有具有约束力的规则均源自核心协议代码库及已部署合约。

## ◆ 2. 设计哲学｜系统原则

### 2.1 机制优于叙事
在 Web3 时代，系统的可持续性由机制决定，而非故事。  
叙事或许能够吸引短期关注，但长期价值留存依赖于：  
→ 可验证的规则  
→ 透明的激励来源  
→ 可预测的资产行为  

NOR Protocol 被设计为一个机制驱动的系统，激励来源于参与规则本身，而非人为分配。  

### 2.2 协议层中立性
NOR Protocol 不依赖于：  
→ 人工审批  
→ 上线后的参数操控  
→ 中心化管理员权限  

协议一经部署，即强制执行：  
→ 统一条件  
→ 自动执行  
→ 非歧视性结果  

所有参与者行为均受代码约束，而非人为判断。

→ 激励随时间与参与度增长  
→ 奖励由协议活动本身生成
